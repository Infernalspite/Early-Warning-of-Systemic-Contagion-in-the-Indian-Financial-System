5 candidates Phase 03 — Model Component Overview

Same X, same y, five different ways of reading it

Every model trains on identical X_train, y_train and is scored on identical X_test, y_test.

i.
Logistic Regression
Baseline

A single linear boundary over the standardized feature matrix. Included deliberately: if the graph network and the LSTM can't beat this, that's a real finding about the data, not a footnote.

ii.
Random Forest
Tabular

200 trees, class-balanced. The standard tabular reference point, and the one that also hands back feature importances — see the leaders below.

iii.
XGBoost
Tabular

Gradient-boosted trees with scale_pos_weight tuned to the ~7% positive rate. Usually the strongest flat-feature model in this class of problem.

iv.
LSTM
Sequential

A 2-layer recurrent network reading 30-trading-day windows of the same features, testing whether the order of recent stress carries signal a flat snapshot throws away.

v.
GNN — GraphSAGE
Topological

Each date is its own graph: 20 banks as nodes, edges where 30-day rolling correlation exceeds 0.4, per-bank features instead of system-wide averages. A pooling layer reduces the graph to one stress score.

Design trade-off, stated rather than hidden: the tabular models above see system-wide averages of the network features; the GNN sees per-bank, node-level graphs. That's the fairer test of whether topology itself helps — but it also means the GNN and the tabular models are technically looking at structurally different views of the same market, which is part of what this comparison is testing.

Read the accuracy column with suspicion. Random Forest, XGBoost and the LSTM all land near 97.3% — which is exactly the size of the negative class in this test window. All three are predicting "normal" on every single day; their real signal only shows up in ROC-AUC (0.79–0.88). Logistic Regression is the only model that actually recalls most crisis days (86.7%), at the cost of a very noisy precision — it wins on F1 by default, not by being a strong classifier. Write-up Phase 05 — Discussion & Limitations

What the table is actually saying

Lead-time vs. raw accuracy

The model that scores best on paper is not the model a risk desk would trust. Random Forest, XGBoost and the LSTM all sit at ~97.3% accuracy by refusing to ever call a crisis — useless as an early-warning system, however clean the headline number looks.

Logistic Regression is the only model with usable recall (86.7%): it catches most pre-crisis windows, early, at the cost of frequent false alarms. The LSTM carries the best ranking quality (ROC-AUC 0.883) without ever crossing its own decision threshold — the signal is there in the probabilities, the cutoff is just miscalibrated for a ~7% positive rate. In a real deployment these point to different thresholds, not different winners.

The GNN is the outlier worth naming directly: ROC-AUC 0.027 is worse than a coin flip, which reads as a genuine negative result rather than noise — with only six labeled crises, the per-bank graphs the GNN needs are the thinnest-fed model in the comparison.

Limitations

  • Six labeled events. Every model is ultimately learning from six historical crises. That's not enough to separate "the model understands systemic stress" from "the model memorized 2018 and 2020."
  • Severe class imbalance. ~7% positive rate in training, closer to 1-in-14 in the test window — exactly why three of five models collapsed to the majority-class strategy.
  • Label noise at the boundary. The 30-day pre-crisis window is a modeling choice, not a fact; days 28–30 before an event look almost identical to normal days.
  • Structurally different inputs. The GNN's node-level view and the tabular models' system-wide averages are not strictly comparable inputs, only comparable outputs.
  • Batch, not streaming, data. Everything above is fit on data pulled and frozen once — see the note on the market pulse strip at the top of this page.
